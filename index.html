Object-Oriented Programming (OOP)

A detailed and exhaustive list of Object-Oriented Programming (OOP) interview questions, covering everything from fundamental concepts to advanced design patterns and real-world applications. This is suitable for freshers and experienced developers alike, especially for roles involving Java, Python, C++, or other OOP-supporting languages.

Core OOP Concepts

1. What is Object-Oriented Programming (OOP)?

Object-Oriented Programming is a programming paradigm based on the concept of "objects," which contain data (attributes) and methods (functions) that operate on the data. It aims to structure software in a way that is modular, reusable, and easy to maintain.

2. What are the benefits of using OOP?

  • Modularity: Code is organized into objects.
  • Reusability: Classes and objects can be reused across applications.
  • Scalability: Easy to expand or modify without affecting other parts.
  • Maintainability: Clean and understandable code structure.
  • Abstraction: Hide internal implementation details.
  • Encapsulation: Protect data from unauthorized access.

3. Compare OOP with procedural programming.

Feature OOP Procedural Programming
Focus Objects and data Functions and procedures
Data Handling Encapsulated within objects Shared across functions
Code Reuse High via inheritance and classes Low
Real-world Mapping Easy Harder

4. List the main principles of OOP.

  • Encapsulation
  • Abstraction
  • Inheritance
  • Polymorphism

5. What is a class? How is it different from an object?

A class is a blueprint or template for creating objects. An object is an instance of a class, containing real values for the attributes defined in the class. 

class Car {
        String color;
        void drive() { System.out.println("Driving..."); }
    }
    
    Car myCar = new Car(); // Object

6. How do you create a class and object in Java/Python/C++?

Java:

class Person {
        String name;
    }
    Person p = new Person();

Python:

class Person:
        def __init__(self, name):
            self.name = name
    
    p = Person("Alice")

C++:

class Person {
    public:
        string name;
    };
    Person p;

7. Can we create a class without any data members?

Yes, a class can contain only methods or even be empty. This is useful for grouping behaviors or providing utility functions.

8. What is an instance?

An instance is a concrete occurrence of any object created from a class. For example, myCar is an instance of the class Car.

9. What is the lifecycle of an object?

  1. Instantiation: Object is created using a class.
  2. Initialization: Constructor initializes the object.
  3. Use: Object performs actions through its methods.
  4. Destruction: Memory is released (garbage collection or destructors).

10. What are instance vs class (static) variables?

  • Instance variable: Tied to each object; each instance has its own copy.
  • Class (static) variable: Shared across all instances of the class. 
class Example {
        static int count;     // class variable
        int value;            // instance variable
    }

Four Pillars of OOP

1. Encapsulation

What is encapsulation and why is it important?

Encapsulation is the concept of wrapping data (variables) and code (methods) together into a single unit — usually a class. It restricts direct access to some components and can prevent the accidental modification of data.

Importance:

  • Protects internal state
  • Promotes modular design
  • Improves code maintainability

How do you achieve encapsulation in Java/Python?

Java:

public class Account {
        private double balance;
    
        public double getBalance() { return balance; }
        public void deposit(double amount) { balance += amount; }
    }

Python:

class Account:
        def __init__(self):
            self.__balance = 0  # private variable
    
        def get_balance(self):
            return self.__balance
    
        def deposit(self, amount):
            self.__balance += amount

What is the benefit of using private variables?

Private variables prevent unauthorized access and modification from outside the class, enforcing data protection.

Give a real-life example of encapsulation.

A bank account: You can deposit or withdraw through methods, but you cannot directly change the balance.

2. Abstraction

What is abstraction in OOP?

Abstraction is the process of hiding complex implementation details and showing only the essential features of an object.

Difference between abstraction and encapsulation

Abstraction Encapsulation
Focuses on hiding implementation Focuses on hiding data
Achieved via interfaces/abstract classes Achieved via access modifiers

How is abstraction implemented in Java/Python?

Java:

abstract class Vehicle {
        abstract void start();
    }

Python:

from abc import ABC, abstractmethod
    
    class Vehicle(ABC):
        @abstractmethod
        def start(self):
            pass

What is an abstract class? Can it have a constructor?

An abstract class is a class that cannot be instantiated and can have abstract and concrete methods. Yes, it can have a constructor to initialize fields.

What are abstract methods?

Abstract methods are methods declared without implementation. Subclasses must override them.

Can we instantiate an abstract class?

No, abstract classes cannot be instantiated directly.

3. Inheritance

What is inheritance and its advantages?

Inheritance is a mechanism where one class acquires the properties and behaviors of another class. It promotes code reusability and a logical class hierarchy.

Types of inheritance

  • Single – one parent, one child
  • Multilevel – inheritance through multiple levels
  • Multiple – one class inherits from multiple classes (supported in Python)
  • Hierarchical – multiple classes inherit from one base class
  • Hybrid – combination of above types

How is inheritance implemented in Java vs Python?

Java:

class Animal {
        void sound() { System.out.println("Animal sound"); }
    }
    
    class Dog extends Animal {
        void bark() { System.out.println("Bark"); }
    }

Python:

class Animal:
        def sound(self):
            print("Animal sound")
    
    class Dog(Animal):
        def bark(self):
            print("Bark")

What is method overriding?

Overriding occurs when a subclass provides a specific implementation of a method already defined in its superclass.

Can constructors be inherited?

Constructors are not inherited, but the child class can call the parent class constructor using super().

Can private members be inherited?

No, private members are not directly inherited.

Use of super keyword in inheritance

super is used to refer to the immediate parent class and is commonly used to call parent constructors or overridden methods.

4. Polymorphism

What is polymorphism?

Polymorphism means "many forms." It allows objects to be treated as instances of their parent class, with behavior that depends on their actual subclass.

Types of polymorphism

  • Compile-time (Static): Method overloading
  • Runtime (Dynamic): Method overriding

What is method overloading and method overriding?

Overloading: Same method name, different parameters (Java only) 

class Display {
        void show(int a) {}
        void show(String s) {}
    }

Overriding: Same method signature, different implementation in subclass 

class Animal {
        void sound() { System.out.println("Sound"); }
    }
    class Dog extends Animal {
        void sound() { System.out.println("Bark"); }
    }

Can constructors be overloaded?

Yes, you can define multiple constructors with different parameters in Java/C++.

Explain operator overloading

Operator overloading lets you redefine the meaning of operators for user-defined types (e.g., +, == in Python/C++).

Python:

class Point:
        def __init__(self, x): self.x = x
        def __add__(self, other): return Point(self.x + other.x)

What is dynamic dispatch?

Dynamic dispatch is the process of selecting which implementation of a polymorphic method to call at runtime based on the object type.

Give real-world examples of polymorphism.

  • A printer can print documents, PDFs, or images – same interface, different behaviors.
  • A remote control for a TV or AC – the button looks same, but behavior changes based on device.

Access Modifiers / Specifiers

1. What are access modifiers in OOP?

Access modifiers (or access specifiers) define the visibility or accessibility of classes, methods, and variables to other classes or packages/modules.

2. Difference between public, private, and protected

Modifier Class Package (Java) / Module (Python) Subclass Outside
public Yes Yes Yes Yes
protected Yes Yes Yes No
private Yes No No No
default (Java only) Yes Yes No No

Java Example:

public class Example {
        public int publicVar = 1;
        protected int protectedVar = 2;
        private int privateVar = 3;
        int defaultVar = 4; // package-private
    }

Python Example:
Python doesn't enforce strict access modifiers, but uses naming conventions: 

class Example:
        def __init__(self):
            self.public_var = 1          # public
            self._protected_var = 2      # protected (by convention)
            self.__private_var = 3       # private (name mangling)
  • __private_var becomes _Example__private_var internally (name mangling).
  • Python relies on developer discipline rather than enforcement.

3. What is default/package-private access in Java?

If no modifier is specified in Java, the member has package-private access — it's accessible only within the same package.

4. How does Python handle access control?

Python does not have true private/protected keywords. Instead:

  • _single_underscore: indicates "protected" (use with caution)
  • __double_underscore: triggers name mangling, simulating "private"

But all attributes can still technically be accessed from outside if needed.

5. Can access modifiers be applied to classes?

In Java:

  • Top-level classes can only be public or default (package-private).
  • Inner classes can use public, protected, private.

In Python:

  • All classes are public; access is controlled via attribute conventions.

Constructors & Destructors

1. What is a constructor?

A constructor is a special method that is automatically called when an object is created. It is used to initialize the object's state (i.e., its fields/attributes).

2. Characteristics of constructors

  • Name is same as the class name (Java, C++).
  • No return type (not even void).
  • Automatically invoked during object creation.

3. Types of constructors in Java

  • Default Constructor: Takes no arguments.
  • Parameterized Constructor: Accepts parameters to initialize the object.
  • Copy Constructor: Creates a new object as a copy of another (common in C++).

Java Example:

class Person {
        String name;
        
        // Default constructor
        Person() {
            name = "Unknown";
        }
    
        // Parameterized constructor
        Person(String name) {
            this.name = name;
        }
    }

4. Constructor in Python

Python Example:

class Person:
        def __init__(self, name="Unknown"):
            self.name = name
  • Python uses __init__() as a constructor.
  • You can have default arguments to simulate constructor overloading.

5. Can constructors be overloaded?

Java: Yes.

Person() {}
    Person(String name) {}

Python: Not directly (no function overloading), but you can use default arguments or *args. 

class Person:
        def __init__(self, name=None):
            self.name = name if name else "Unknown"

6. What is constructor chaining?

Constructor chaining is when one constructor calls another constructor in the same class or the parent class.

Java:

class Person {
        String name;
    
        Person() {
            this("Default");
        }
    
        Person(String name) {
            this.name = name;
        }
    }

Python: Can use __init__ with logic, or call parent constructor using super().

7. What is a destructor?

A destructor is a special method called when an object is destroyed to release resources.

Java: No explicit destructor. JVM garbage collector handles it. You can override finalize() (deprecated in modern Java). 

protected void finalize() {
        System.out.println("Object is being destroyed");
    }

Python:

class Demo:
        def __del__(self):
            print("Destructor called")

8. Can we overload destructors?

No, neither in Java nor Python. There's only one destructor per class.

9. What is the difference between constructor and method?

Feature Constructor Method
Purpose Initializes object Defines behavior
Name Same as class (Java) / __init__ Any valid identifier
Return type None Can have return value
Invocation Called automatically during object creation Called manually

10. Can constructors be private?

Java: Yes, used in Singleton pattern or factory methods. 

class Singleton {
        private static Singleton instance = new Singleton();
        private Singleton() {}
        public static Singleton getInstance() {
            return instance;
        }
    }

Python: No enforced private constructors, but you can simulate with metaclasses or naming conventions.

Keywords Related to OOP

1. What is the this keyword?

Java:
this refers to the current object — used to access instance variables and methods from within the class. 

class Person {
        String name;
        Person(String name) {
            this.name = name; // refers to instance variable
        }
    }

Python:
Python uses self in place of this. It's passed explicitly to instance methods. 

class Person:
        def __init__(self, name):
            self.name = name  # self refers to current instance

2. What is the super keyword?

Java:
super refers to the parent class. It's used to:

  • Access parent methods/variables
  • Call parent constructor
class Animal {
        void sound() { System.out.println("Animal sound"); }
    }
    class Dog extends Animal {
        void sound() {
            super.sound(); // call parent method
            System.out.println("Bark");
        }
    }

Python:
Python uses super() similarly to call parent methods or constructors. 

class Animal:
        def __init__(self): print("Animal constructor")
    
    class Dog(Animal):
        def __init__(self):
            super().__init__()  # calls parent constructor
            print("Dog constructor")

3. What is the final keyword in Java?

final is used to restrict modification.

  • final variable → cannot be reassigned.
  • final method → cannot be overridden.
  • final class → cannot be inherited.
final int x = 10;
    
    final class A {} // Can't extend A

4. What is the static keyword?

  • Belongs to the class rather than an instance.
  • Can be accessed without creating an object.
  • Used for variables, methods, and blocks.
class Example {
        static int count = 0;
        static void display() {
            System.out.println("Static method");
        }
    }

Python equivalent:

class Example:
        count = 0
    
        @staticmethod
        def display():
            print("Static method")

5. What is the difference between this and super?

Keyword Refers to Used for
this Current class instance Accessing members of the current class
super Parent class instance Accessing members of the parent class

6. Can a static method access instance variables?

No. Static methods belong to the class, and they don't have access to this or instance members unless an object is passed explicitly.

7. Can we override static methods?

No. Static methods are not polymorphic and are resolved at compile time in Java.

Interfaces & Abstract Classes

1. What is an abstract class?

An abstract class is a class that cannot be instantiated and may contain abstract methods (methods without a body) as well as concrete methods (with a body).

2. What is an interface?

An interface is a contract that a class must follow. It only contains method declarations (without implementation) and constants (in Java).

  • A class implements an interface.
  • Promotes multiple inheritance of type.

3. Difference between abstract class and interface

Feature Abstract Class Interface
Methods Can have both abstract and concrete Only abstract (Java 7), default/static from Java 8
Variables Instance and static variables allowed Only public static final constants
Inheritance extends (single inheritance) implements (multiple inheritance)
Constructors Yes No
Instantiation Cannot instantiate Cannot instantiate

4. Can we have method implementations in interfaces?

  • Java:
    • Java 8 introduced default and static methods with implementation.
    • Java 9+ allows private methods in interfaces. 
interface A {
      default void show() {
          System.out.println("Default method in interface");
      }
  }
  • Python: Python does not have interfaces built-in like Java, but it uses abstract base classes (abc module).

5. Abstract class vs Interface in Python

Python allows abstract classes using abc module.

from abc import ABC, abstractmethod
  
  class Shape(ABC):
      @abstractmethod
      def area(self): pass

To mimic interface behavior, define an abstract class with only abstract methods.

6. Can an abstract class have a constructor?

Yes. An abstract class can have a constructor to initialize data members, and it is called when a subclass is instantiated.

Java:

abstract class Animal {
      Animal() {
          System.out.println("Animal constructor");
      }
  }

Python:

class Animal(ABC):
      def __init__(self):
          print("Animal constructor")

7. Can a class implement multiple interfaces?

Java: Yes. A class can implement multiple interfaces. 

interface A {}
  interface B {}
  class C implements A, B {}

Python: Supports multiple inheritance directly.

8. Can an interface extend another interface?

Java: Yes. An interface can extend one or more interfaces. 

interface A {}
  interface B extends A {}

9. Why use interfaces?

  • Supports polymorphism
  • Promotes loose coupling
  • Enables multiple inheritance
  • Defines APIs/contracts for external systems

10. Can a class be abstract without abstract methods?

Yes. You can declare a class abstract even if it has no abstract methods, to prevent instantiation. 

abstract class Animal {
      void eat() {
          System.out.println("Eat");
      }
  }

Class Relationships

1. What is Association in OOP?

Association represents a relationship between two independent classes. One class uses or is aware of another.

  • Example: A Teacher works in a School.
  • It can be one-to-one, one-to-many, many-to-one, or many-to-many.

Java Example:

class Teacher {
        String name;
    }
    
    class School {
        Teacher teacher; // Association
    }

Python Example:

class Teacher:
        def __init__(self, name):
            self.name = name
    
    class School:
        def __init__(self, teacher):
            self.teacher = teacher  # Association

2. What is Aggregation in OOP?

Aggregation is a "has-a" relationship, where one class contains another but both can exist independently.

  • Example: A Department has Professors, but if the department is deleted, professors may still exist.
  • Represented as a weaker form of composition.

Java Example:

class Professor {
        String name;
    }
    
    class Department {
        List<Professor> professors; // Aggregation
    
        Department(List<Professor> professors) {
            this.professors = professors;
        }
    }

Python Example:

class Professor:
        def __init__(self, name):
            self.name = name
    
    class Department:
        def __init__(self, professors):
            self.professors = professors

3. What is Composition in OOP?

Composition is a stronger form of Aggregation. If the container object is destroyed, the contained objects also get destroyed.

  • Example: A House has Rooms. If the house is destroyed, rooms are too.

Java Example:

class Room {
        String name;
        Room(String name) {
            this.name = name;
        }
    }
    
    class House {
        Room room;
    
        House() {
            room = new Room("Bedroom"); // Strong ownership
        }
    }

Python Example:

class Room:
        def __init__(self, name):
            self.name = name
    
    class House:
        def __init__(self):
            self.room = Room("Bedroom")  # Strong containment

4. Difference between Association, Aggregation, and Composition

Feature Association Aggregation Composition
Relationship General (uses-a) Has-a (weak) Has-a (strong)
Dependency Low Low High
Lifetime Independent Independent Dependent (child dies with parent)
Example Student–Library Department–Professor House–Room

5. How to choose between aggregation and composition?

  • Use aggregation when the contained object can outlive the container.
  • Use composition when the contained object should not exist without the container.

Polymorphism

1. What is polymorphism in OOP?

Polymorphism means "many forms." It allows objects to behave differently based on their context or data types.

Two main types:

  • Compile-time polymorphism (Method Overloading)
  • Runtime polymorphism (Method Overriding)

2. What is method overloading?

Method Overloading = Same method name, but different parameters (number or type) in the same class.

Java Example:

class Calculator {
      int add(int a, int b) {
          return a + b;
      }
  
      double add(double a, double b) {
          return a + b;
      }
  }

Python Example:
Python doesn't support true method overloading; use default arguments or *args. 

class Calculator:
      def add(self, a, b=0):
          return a + b

Or with @singledispatch from functools for function overloading based on type.

3. What is method overriding?

Method Overriding = Subclass provides a new implementation of a method inherited from a superclass.

Java Example:

class Animal {
      void sound() {
          System.out.println("Animal sound");
      }
  }
  
  class Dog extends Animal {
      @Override
      void sound() {
          System.out.println("Bark");
      }
  }

Python Example:

class Animal:
      def sound(self):
          print("Animal sound")
  
  class Dog(Animal):
      def sound(self):
          print("Bark")

4. Difference between overloading and overriding

Feature Overloading Overriding
Occurs in Same class Parent-child classes
Parameters Must differ Must be same
Return type Can differ Should be same or covariant (Java)
Binding time Compile-time Runtime

5. Can we override static methods?

No. Static methods are not polymorphic. You can hide them in Java, but not override.

6. Can constructors be overridden?

No. Constructors are not inherited, hence cannot be overridden.

7. Can we overload constructors?

Yes. Constructor overloading is common.

Java:

class Person {
      Person() {}
      Person(String name) {}
  }

Python:
Not directly. Use default values or *args: 

class Person:
      def __init__(self, name=None):
          self.name = name or "Unknown"

8. Can we override private or final methods in Java?

  • Private methods: No (they are not inherited).
  • Final methods: No (cannot be overridden).

9. What is dynamic method dispatch?

Dynamic Method Dispatch is Java's runtime polymorphism where method calls are resolved at runtime based on the actual object type. 

Animal a = new Dog();  // Upcasting
  a.sound(); // Bark

Exception Handling in OOP

1. What is exception handling in OOP?

Exception handling is a mechanism to handle runtime errors gracefully, without crashing the program.

It promotes:

  • Robustness
  • Maintainability
  • Separation of error-handling logic from business logic

2. What is the difference between error and exception?

Feature Error Exception
Type Severe issue, typically unrecoverable Logical issue, can be handled
Examples OutOfMemoryError, StackOverflowError NullPointerException, IOError
Handling Cannot be handled normally Can be handled using try-catch

3. What are the types of exceptions in Java?

  1. Checked Exceptions: Checked at compile time (e.g., IOException, SQLException)
  2. Unchecked Exceptions: Occur at runtime (e.g., NullPointerException, ArithmeticException)

4. Exception hierarchy in Java

Object  
  └── Throwable  
        ├── Error  
        └── Exception  
              ├── RuntimeException  
              └── Other checked exceptions

5. What is the try-catch-finally block?

Used to handle exceptions:

try {
    // code that may throw an exception
} catch (ExceptionType e) {
    // handler
} finally {
    // cleanup code (always executes)
}

Python Equivalent:

try:
    # risky code
except Exception as e:
    # handle exception
finally:
    # always run this block

6. What is the finally block?

  • Executes regardless of whether an exception was thrown or caught.
  • Used for resource cleanup (closing files, releasing DB connections).

7. What is the use of throw and throws in Java?

  • throw: Used to manually throw an exception.
  • throws: Declares exceptions a method can throw.
void checkAge(int age) throws Exception {
    if (age < 18) throw new Exception("Underage");
}

8. Can we have multiple catch blocks?

Yes. Java allows multiple catch blocks to handle different exceptions specifically. 

try {
    // risky code
} catch (IOException e) {
    // handle IO
} catch (Exception e) {
    // handle others
}

9. Can we catch multiple exceptions in one block?

Yes, since Java 7:

catch (IOException | SQLException e) {
    // common handler
}

10. How do you create a custom exception?

Java:

class MyException extends Exception {
    MyException(String message) {
        super(message);
    }
}

Python:

class MyException(Exception):
    def __init__(self, message):
        super().__init__(message)

11. What is exception propagation?

If a method doesn't catch an exception, it is passed to the calling method, continuing up the call stack.

Miscellaneous QnA

1. What is object cloning?

Object cloning creates an exact copy of an object with the same values.

Java Example:
Java provides Cloneable interface and clone() method: 

class Student implements Cloneable {
  int id;
  String name;

  public Student clone() throws CloneNotSupportedException {
      return (Student) super.clone();
  }
}
  • Must implement Cloneable, otherwise CloneNotSupportedException.

Python Example:

import copy

original = {"name": "John"}
clone = copy.copy(original)        # Shallow copy
deep_clone = copy.deepcopy(original)  # Deep copy

2. What is a marker interface?

A marker interface is an empty interface used to mark a class with a capability.

Examples: Serializable, Cloneable in Java. 

class Student implements Serializable {}

Python uses decorators or metadata instead.

3. What is immutability in OOP?

An immutable object cannot be modified after creation.

Java Example (Immutable class):

final class Person {
  private final String name;

  Person(String name) {
      this.name = name;
  }

  public String getName() {
      return name;
  }
}

Python Example:

from dataclasses import dataclass

@dataclass(frozen=True)
class Person:
  name: str

4. What is encapsulation vs immutability?

Concept Encapsulation Immutability
Purpose Hides internal data Prevents modification
Access Via getters/setters Only via getters
Can change? Yes, if allowed No

5. What is data hiding?

Data hiding refers to restricting access to internal object data by making fields private and exposing only necessary APIs.

Java:

class Account {
  private double balance;

  public double getBalance() {
      return balance;
  }
}

Python:

class Account:
  def __init__(self):
      self.__balance = 0  # double underscore for name mangling

6. What is cohesion and coupling?

  • Cohesion: Degree to which class members are related to each other.
    • High cohesion = Good design
  • Coupling: Degree of dependency between modules/classes.
    • Low coupling = Better modularity

7. What is a singleton class?

A singleton ensures only one instance of a class exists during runtime.

Java Example:

class Singleton {
  private static Singleton instance = new Singleton();
  private Singleton() {}
  public static Singleton getInstance() {
      return instance;
  }
}

Python Example:

class Singleton:
  _instance = None
  def __new__(cls):
      if cls._instance is None:
          cls._instance = super().__new__(cls)
      return cls._instance

8. What is the difference between deep copy and shallow copy?

Copy Type Copies References? Copies Nested Objects?
Shallow Copy Yes No
Deep Copy No Yes

9. What is object slicing? (Java-specific)

Object slicing occurs when a subclass object is assigned to a superclass reference, and the subclass-specific fields/methods are sliced off (not accessible). 

class A { int x; }
class B extends A { int y; }

A obj = new B(); // obj.y not accessible

Not applicable in Python due to dynamic typing.

10. What is the difference between instance variable and class variable?

Feature Instance Variable Class Variable
Storage Per object Shared across all objects
Defined using No static keyword (Java) static keyword (Java)

Java:

class MyClass {
  int x;           // instance
  static int y;    // class
}

Python:

class MyClass:
  x = 5       # class variable

  def __init__(self):
      self.y = 10  # instance variable
